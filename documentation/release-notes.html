kryten-2.1.2

kryten terminates if the configuration file does not specify any PVs.

In verbose mode kryten outputs its interpretation of the configuration.

This version is more tolerant when parsing te configuration file. White space is now allowed between the sub-fields of the match criteria. # White space now allowed -----+--+---+-+----+-+ # | | | | | | # v v v v v v SR11BCM01:LIFETIME_MONITOR -1.0 ~ 5.0 | 50.0 ~ 100.0 echo # Whereas previosly had to be like this: # SR11BCM01:LIFETIME_MONITOR -1.0~5.0|50.0~100.0 echo # end

An optional index values may now be specified (for waveform and other array PVs). The default is 1, the first element (kryten numbers array elements from 1 to N as oppossed to C style 0 to N-1). If there are less elements in the array than the specified index value then the PV is ignored. Example: # Monior the third element of the array for being prime less than 32. # SOME_WAVEFORM:ARRAY [3] 2 ~ 3 | 5 | 7 | 11 | 13 | 17 | 19 | 23 | 31 /home/user/bin/prime.sh

The command script or program should now expect four parameters - the (new) 4th parameter is the element index.
